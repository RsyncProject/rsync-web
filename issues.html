Current issues

  1. Q: Transfers die with something like 
    rsync: error writing 4 unbuffered bytes - exiting: Broken pipe
rsync error: error in rsync protocol data stream (code 12) at ../src/io.c(463)

    A: Sometimes fatal error messages from the remote machine can be lost on the way to the client; you should investigate on the server for something stopping the transmission. The most common cause is that the destination disk is full.

  2. Q: interrupted transfers with --partial are not continued properly; rsync always transfers the whole file.

    A: 2.5.4 had a bug that made it act as if --whole-file was always set. Please upgrade to 2.5.5.
