Current issues

  1. Q: Transfers die with something like 

    rsync: error writing 4 unbuffered bytes - exiting: Broken pipe
rsync error: error in rsync protocol data stream (code 12) at ../src/io.c(463)

    A: Sometimes fatal error messages from the remote machine can be lost on the way to the client; you should investigate on the server for something stopping the transmission. Several common causes:

    • The destination disk is full.
    • The remote rsync executable wasn't found.
    • Your remote-shell setup isn't working right or isn't "clean" (i.e. is sending spurious text to rsync).

  2. Q: Using --dry-run (-n) doesn't show all changed directories.

    A: Correct. Rsync shows all other changes, but some directory changes are not currently described in --dry-run mode. Hopefully a future version will fix this.
