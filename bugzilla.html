rsync bug-tracking

If you think you have found a bug in rsync we would appreciate it if you would check the list of current issues and search the bugzilla database to see if the problem is already known. Also, there are several patches for being-considered features in the patches dir.

Also, please take a couple of minutes to read Simon Tatham's advice on how to report bugs.

To report a bug or make suggestions, use one of these methods:

  • The preferred method is to create a report via bugzilla (this does require that you have created a bugzilla account, but that is a very simple process). All new items and changes to existing items get sent as email to the mailing list, so people naturally stay up-to-date with what is happening in the bug-tracking database. (N.B. if you're making a suggestion, please be sure to set the Severity to Enhancement.)
  • Alternately, you can send an email to the rsync mailing list, which is at rsync@lists.samba.org, especially if you wish to discuss a bug before making a report. While the mailing list accepts patches (typically as MIME attachements), it is often easier to attach your patch to the appropriate bug report in bugzilla.

Thanks for helping out!
